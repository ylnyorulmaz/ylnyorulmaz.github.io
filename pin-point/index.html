📍 PinPoint

A privacy-first map tool for saving places that matter.

PinPoint is a local-first Progressive Web App built for people who want to save meaningful coordinates without giving location history to cloud platforms.

It strips map usage down to the essentials: drop a pin, label it, find it again — privately.

Overview

PinPoint was designed as a focused alternative to feature-heavy mapping ecosystems. No accounts, no recommendations, no ads, and no behavioral profiling.

The core idea is intentionally narrow: let users mark personal places and come back to them quickly.

The use case

  • Where did I park?
  • What was that quiet fishing spot?
  • Where exactly did we take that photo?
  • Where did I propose?

The Problem

Modern map apps are powerful, but many are optimized for engagement rather than ownership.

  • They require accounts.
  • They store location history in the cloud.
  • They blend personal memories with commercial recommendations.

Core Principles

  1. Local-first: Pins are stored in local storage on-device.
  2. Minimal surface area: No ratings, reviews, ads, or recommendation loops.
  3. Utility over complexity: Precision placement and fast retrieval first.

Feature Highlights

  • Precision pin dropping: Tap to draft, drag to fine-tune coordinates.
  • Category system: 10 visual categories with custom icons and colors.
  • Map layers: Standard, dark mode, and satellite options.
  • Smart directions: One-tap handoff to Google Maps navigation.
  • List tools: Live search, filters, and mobile-friendly swipe-to-delete.
  • PWA support: Home-screen install and fullscreen app-like usage.

Technical Architecture

  • Frontend: React 19, Vite, TypeScript, Tailwind CSS.
  • Maps: Leaflet with OSM, Carto, and Esri raster tiles.
  • State: LocalStorage with JSON serialization and safe parsing.
  • UX: Mobile-first layouts, safe-area handling, and custom transitions.
  • Delivery: No backend and no service worker (intentional early-stage tradeoff).

Tradeoffs & Constraints

PinPoint intentionally avoids accounts, sync, and server infrastructure. The product remains private and simple, but it also means:

  • Clearing browser data deletes saved pins.
  • There is no automatic backup yet.
  • It is not built for collaborative map editing.

What I Learned

  • Local-first architecture removes ops overhead but raises UX responsibility around data safety.
  • Mobile map UX depends heavily on spacing, safe areas, and gesture expectations.
  • Service workers are powerful but should be introduced only with strict cache discipline.
  • The smallest feature sets are often the hardest to refine.

Future Directions

  • Optional encrypted cloud backup.
  • Shareable view-only maps.
  • Folder-based pin collections.
  • A lightweight opt-in sync mode.
